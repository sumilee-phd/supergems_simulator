# 슈퍼젬스 × Opal 인터랙티브 가이드

> Google Gemini의 **Super Gems**와 **Opal 에디터**를 처음 만나는 분을 위한,
> 클릭 한 번으로 따라 해 보는 **싱글 페이지 시뮬레이터**입니다.
> 설치도, 회원가입도, 환경설정도 필요 없습니다. 브라우저만 있으면 됩니다.

---

## 🚀 1분 만에 시작하기

### 방법 ① 가장 쉬운 방법 — 바로 열기

1. 위쪽 파일 목록에서 [`supergems_simulator.html`](supergems_simulator.html) 을 클릭합니다.
2. 우측 상단의 **Raw** 버튼을 누르거나, 아래 GitHub Pages 링크를 누릅니다.
   👉 https://sumilee-phd.github.io/supergems_simulator/
3. 페이지가 열리면 **튜토리얼 카드가 자동으로 나타납니다.** "다음 →" 만 계속 누르세요.

### 방법 ② 내 컴퓨터에 다운받아 열기

```bash
# 1. 파일 다운로드
git clone https://github.com/sumilee-phd/supergems_simulator.git

# 2. 폴더 이동
cd supergems_simulator

# 3. 브라우저로 더블클릭해서 열기
#    Windows: index.html 더블클릭
#    Mac:     open index.html
```

> 💡 인터넷이 안 되어도 동작합니다. 외부 서버 호출이 전혀 없어요.

## 📖 이 시뮬레이터로 무엇을 배우나요?

Google이 새롭게 공개한 **Super Gems**(자연어로 만드는 AI 미니앱)와
**Opal 에디터**(노드 기반 워크플로우 에디터)의 흐름을
**실제 화면과 똑같이** 따라가며 익힐 수 있습니다.

| 학습 단계 | 무엇을 익히나요? |
|---|---|
| **1단계 · Gems 페이지** | 자연어로 "내가 만들고 싶은 앱"을 묘사하는 법 |
| **2단계 · Opal 에디터** | 노란/파랑/초록 노드의 의미와 `@참조` 사용법 |
| **3단계 · 노드 설정** | User Input → Generate → Output 프롬프트 작성 |
| **4단계 · App 미리보기** | 만든 앱을 실제로 실행해 결과 확인 |
| **5단계 · 실제 배포** | 생성된 프롬프트를 진짜 Gemini에 옮기는 방법 |

## 🎬 사용 흐름 — 화면별 가이드

### 화면 ① Gems 페이지 (시작 화면)

가장 처음 만나는 페이지입니다.

```
┌─────────────────────────────────────────────┐
│  [Gems] [Editor] [App]      Publish · Share │
├─────────────────────────────────────────────┤
│                                             │
│   Build the AI mini-app your team needs.    │
│                                             │
│   [🎯][🍳][📋][💼]   ← 4개 예시 카드          │
│                                             │
│   [____내용을 입력_____________________] [→]  │
└─────────────────────────────────────────────┘
```

**할 일**:
1. 4개 예시 카드 중 하나를 **클릭**하면 입력창에 자동으로 채워집니다.
2. 또는 직접 입력 — 예: `유튜브 링크를 넣으면 블로그 글을 자동으로 만들어 줘`
3. 오른쪽의 **빨간 화살표 버튼**을 누르면 Gemini가 워크플로우를 분석합니다.
4. "Opal 에디터에서 편집" 버튼을 눌러 다음 화면으로 이동합니다.

---

### 화면 ② Opal 에디터 (노드 기반 편집)

3개의 색깔 있는 노드가 화살표로 연결된 화면을 보게 됩니다.

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 🟡 INPUT │ →  │ 🔵 GEN   │ →  │ 🟢 OUT   │
│ User     │    │ Generate │    │ Output   │
└──────────┘    └──────────┘    └──────────┘
   사용자 입력      AI 생성          결과 출력
```

**색깔의 의미**:
- 🟡 **노랑** — 사용자가 무엇을 입력해야 하는지 안내하는 노드
- 🔵 **파랑** — Gemini가 어떻게 분석/생성할지 지시하는 노드 (가장 중요!)
- 🟢 **초록** — 결과를 어떤 형식으로 보여줄지 정하는 노드

**할 일**:
1. **노드를 클릭** — 우측 사이드바에 해당 노드의 설정이 나타납니다.
2. 사이드바 우상단의 **✏️ 아이콘**을 누르면 편집 모드로 전환됩니다.
3. 프롬프트를 수정한 뒤 하단의 **변경사항 저장** 버튼을 누르세요.
4. `@User Input` 처럼 **@ 기호**로 앞 노드의 출력을 참조할 수 있습니다.

> 💡 **Advanced Settings**에서 시스템 프롬프트와 Temperature를 조정할 수 있습니다.

---

### 화면 ③ App 미리보기

만든 앱이 실제로 어떻게 동작하는지 시뮬레이션합니다.

```
┌─────────────────────────────────────────┐
│            — Live Preview —             │
│                                         │
│        나의 AI 자동화 앱                 │
│                                         │
│   [____내용 입력_____________________]   │
│                                         │
│   [        실행하기 (빨강 버튼)        ]  │
│                                         │
│   [ AI 분석 결과가 여기에 출력됩니다 ]    │
└─────────────────────────────────────────┘
```

**할 일**:
1. 텍스트박스에 아무 내용이나 입력합니다.
2. **실행하기** 버튼을 누르면 가짜 AI 결과가 표시됩니다 (시뮬레이션).
3. 완료 배너가 뜨면 **"실제 슈퍼젬스 만들기 →"** 를 눌러 진짜 Gemini로 이동합니다.

## 🛠️ 진짜 Super Gems로 옮기기

시뮬레이터로 연습이 끝났다면, 실제 Gemini에서 똑같이 만들어 봅시다.

```
┌────────────────────────────────────────────┐
│ STEP 1 → STEP 2 → STEP 3 → STEP 4 → STEP 5 │
│  접속      입력      편집      저장      배포  │
└────────────────────────────────────────────┘
```

1. **접속** — https://gemini.google.com/gem-labs/new 으로 이동
2. **입력** — 시뮬레이터에서 만든 앱 설명을 그대로 붙여넣기
3. **편집** — 생성된 단계에서 **"Open Advanced Editor"** 클릭 → Opal 진입
4. **각 노드 설정** — 시뮬레이터의 사이드바에서 만든 프롬프트를 복사 → 붙여넣기
5. **저장 & 배포** — 우측 상단 **Publish** → 공유 링크가 만들어집니다

> ✨ **꿀팁**: 시뮬레이터의 **"📋 프롬프트 전체 보기"** 모달에서
> 3개 노드의 프롬프트를 한 번에 복사할 수 있습니다.

## ❓ 자주 묻는 질문

<details>
<summary><b>Q. 이 페이지는 진짜 Gemini와 연결되나요?</b></summary>

아니요. 이 시뮬레이터는 **UI 학습 목적**의 정적 HTML이며,
실제 AI 호출은 일어나지 않습니다. 출력되는 결과는 미리 정해진 템플릿입니다.
실제로 사용하려면 위의 **"진짜 Super Gems로 옮기기"** 단계를 따라 주세요.
</details>

<details>
<summary><b>Q. 모바일에서도 동작하나요?</b></summary>

기본적인 화면 전환은 동작합니다. 다만 Opal 에디터의 노드 캔버스는
**데스크탑 화면(1024px 이상)**에 최적화되어 있습니다.
가능하면 노트북·데스크탑에서 사용해 주세요.
</details>

<details>
<summary><b>Q. 영문/번역본이 있나요?</b></summary>

현재는 한국어 버전만 제공됩니다. 영문화 PR은 환영합니다!
</details>

<details>
<summary><b>Q. 색깔별 노드가 의미하는 게 뭐예요?</b></summary>

Opal 에디터의 표준 컬러 컨벤션을 따랐습니다.
- 🟡 노랑 = 입력(Input)
- 🔵 파랑 = 처리/생성(Process)
- 🟢 초록 = 출력(Output)
</details>
